Important FAQ

What is the document object model (DOM)?

The Document Object Model is a cross-platform and language-independent interface that treats an HTML or XML document as a tree structure wherein each node is an object representing a part of the document. The DOM represents a document with a logical tree.

How do you select element from DOM?

Since element IDs are required to be unique if specified, they're a useful way to get access to a specific element quickly. If you need to get access to an element which doesn't have an ID, you can use querySelector() to find the element using any selector.

What is event delegation in the context of the DOM, and why is it helpful?

Event delegation is a technique in the DOM where a single event listener is added to a parent element to manage events for multiple child elements. Instead of adding individual listeners to each child, the parent element listens for events that bubble up from its children, and then determines which child triggered the event.

Why is it helpful?
Improves performance by reducing the number of event listeners. Simplifies code, especially when managing many dynamic elements. Easier maintenance, as new child elements can be managed without adding extra listeners. Event delegation leverages event bubbling, where events propagate from the target element up to the parent elements.

How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes and styles using the DOM:

Select the Element:
Use methods like document.getElementById() or document.querySelector(). Change Attributes:
Use element.setAttribute('attributeName', 'value') to set an attribute.
Use element.getAttribute('attributeName') to get an attribute.
Use element.removeAttribute('attributeName') to remove an attribute.

Change Styles:
Use element.style.propertyName = 'value' to set a style.
Example: element.style.backgroundColor = 'red'.
These methods let you dynamically modify elements in the document.
